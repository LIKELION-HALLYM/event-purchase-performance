# 환희 - 작업 내용 정리

## 담당 사이트
- **지그재그 (Zigzag)** - https://zigzag.kr

## 작업 내용

### 1. 데이터 크롤링 (`ziguzegu.py`)
- 지그재그 GraphQL API (`GetPageInfoForWeb`) 활용
- 수집 필드: 상품명, 현재가, 원래가, 할인율, 쇼핑몰, 평점, 리뷰수, 카테고리, 상품링크, 이미지
- 페이지네이션(`end_cursor`) 자동 처리
- 결과: `zigzag_final_list.csv` (180개)

### 2. 테스트 데이터 생성 (`generate_data.py`)
- 실제 크롤링 데이터의 분포(가격대, 쇼핑몰명, 카테고리, 평점 등) 기반으로 10만건 생성
- 결과: `zigzag_100k.csv` (100,000개)

### 3. DB 성능 벤치마크 (`benchmark.py`)
- 대상 DB: MySQL, PostgreSQL, MongoDB
- 테스트 항목: 전체 조회 / 카테고리 필터 / 가격 범위 조회 / 평점 TOP100 / 상품명 검색
- 방식: 웜업 1회 제외, 5회 반복 측정 → 평균/최소/최대
- 결과: `benchmark_result.csv`

## 벤치마크 결과 요약 (10만건 기준, 5회 평균)

| 테스트 | MySQL | PostgreSQL | MongoDB |
|--------|-------|-----------|---------|
| 전체 조회 | 0.8133s | **0.0510s** | 0.3106s |
| 카테고리 필터 | 0.1224s | **0.0094s** | 0.0527s |
| 가격 범위 조회 | 0.5474s | **0.0354s** | 0.2576s |
| 평점 TOP100 | 0.0116s | **0.0010s** | 0.0013s |
| 상품명 검색 | 0.0711s | **0.0146s** | 0.0850s |

## 결론
- **PostgreSQL** 전 항목 1위
- 조회 성능 압도적 + 트랜잭션(결제/장바구니) 안정성 우수
- 본 프로젝트 DB로 **PostgreSQL 추천**
